# Messages

Messages are the primary means of communication in the AO network. They carry data between processes, from users to processes, and from processes back to users. Understanding messages is essential for working with the AO network.

#### What is a Message?

A message in AO is a data structure that:

- **Transmits Information**: Carries data from sender to recipient
- **Triggers Actions**: Causes processes to execute specific handlers
- **Creates Results**: Generates output and potentially new messages
- **Persists Forever**: Stored permanently on Arweave
- **Ordered**: Delivered in sequence by the scheduler

#### Message Structure

Every message contains several components:

**Core Fields:**

- **ID**: Unique transaction identifier (43-character Arweave ID)
- **From**: Sender address (wallet or process)
- **To**: Recipient address (process)
- **Owner**: Arweave wallet that signed the message
- **Timestamp**: When the message was created
- **Block Height**: Arweave block number

**Tags:**

- **Data-Protocol**: "ao" - identifies AO messages
- **Type**: "Message" - distinguishes from processes
- **Variant**: Network variant (ao.N.1, ao.TN.1)
- **Action**: What the message requests (Eval, Transfer, etc.)
- **Additional Tags**: Action-specific parameters

**Data:**

- **Input**: Message payload (Lua code, JSON data, etc.)
- **Output**: Process response to the message
- **Results**: Messages generated by this message

#### Message Flow

**Sending a Message:**

1. User composes message with action and data
2. Wallet signs the message transaction
3. Message is submitted to the network
4. Scheduler receives and queues message
5. Process executes handler for action
6. Process updates state
7. Process may generate result messages
8. Output is stored on Arweave

**Message Types:**

**Incoming Messages:**

- Messages sent to a process
- Trigger process handlers
- Can modify process state
- Visible in process message list

**Outgoing Messages:**

- Messages sent by a process
- Result of process execution
- Can target other processes
- Represent process actions

**Resulting Messages:**

- Generated during message execution
- Spawned by parent message
- Carry computed results
- Form message chains

#### Common Message Actions

**Eval:**

- Execute arbitrary Lua code
- Used for queries and commands
- Can read or modify state
- Most flexible action

**Example:**

```lua
return "Hello from process"
```

**Info:**

- Request process information
- Returns metadata about process
- Standard query action
- No state modification

**Balance:**

- Query token balance
- For specific address or total
- Returns numeric value
- Read-only operation

**Example tags:**

```
Action: Balance
Target: [wallet-address]
```

**Transfer:**

- Send tokens to recipient
- Modifies balances
- Generates Debit-Notice and Credit-Notice
- Requires sufficient balance

**Example tags:**

```
Action: Transfer
Recipient: [wallet-address]
Quantity: 1000000000000
```

**Debit-Notice:**

- Notification of tokens sent
- Auto-generated by Transfer
- Sent to sender's wallet process
- Records outgoing transfer

**Credit-Notice:**

- Notification of tokens received
- Auto-generated by Transfer
- Sent to recipient's wallet process
- Records incoming transfer

#### Message Tags

Tags carry additional information about the message:

**Standard Tags:**

- **Action**: Primary action to execute
- **Recipient**: For transfers
- **Quantity**: Amount for transfers
- **Data**: Additional payload
- **Target**: Query target for reads
- **Ref\_**: Reference to another message

**Custom Tags:**

- Process-specific parameters
- Application metadata
- Routing information
- Protocol extensions

#### Inspecting Messages in Lunar

**Message Info Panel:**

- View From, To, Owner addresses
- See Action and Variant
- Check timestamp and block height
- Copy message ID

**Message Input:**

- View original message data
- See Lua code for Eval actions
- Inspect JSON payloads
- Copy input content

**Message Output:**

- View process response
- See execution results
- Check for errors
- Formatted JSON display

**Resulting Messages:**

- Messages spawned by this message
- Debit and Credit notices
- Process-to-process communication
- Expandable message tree

#### Message Filtering

Lunar provides powerful filtering for message lists:

**Filter by Action:**

- Select from standard actions (Eval, Transfer, etc.)
- Add custom action filters
- Combine multiple actions
- Save action preferences

**Filter by Participant:**

- Recipient: Messages sent to specific address
- Sender: Messages from specific address
- Supports wallet and process addresses
- Copy/paste addresses easily

**Filter by Date:**

- Start date: Messages after specific date
- End date: Messages before specific date
- Date range selection
- Calendar picker interface

**Results Per Page:**

- Configure pagination size
- Options: 10, 25, 50, 100 messages
- Persist preference
- Navigate with next/previous

#### Message Results and Output

**Success Output:**

- JSON formatted response
- Lua return values
- Process state changes
- Generated messages

**Error Output:**

- Error messages and stack traces
- Failed handler execution
- Invalid parameters
- Insufficient balance

**Empty Output:**

- Handler completed with no return
- Silent execution
- State modified without output
- Check resulting messages

#### Best Practices

**Sending Messages:**

- Always verify recipient address
- Check action format in process docs
- Include all required tags
- Test with small amounts for transfers

**Reading Messages:**

- Use filters to find specific messages
- Check timestamps for chronological context
- Inspect both input and output
- Follow resulting message chains

**Error Handling:**

- Read error messages carefully
- Check process source for requirements
- Verify action spelling and capitalization
- Ensure wallet has sufficient balance

**Message Analysis:**

- Compare input vs output
- Track message chains through resulting messages
- Identify patterns in process behavior
- Monitor for unusual activity

#### Message Lifecycle

**Pending:**

- Message submitted but not processed
- Waiting in scheduler queue
- Not yet on Arweave
- May take a few seconds

**Processing:**

- Message being executed by process
- Handler running
- State being updated
- Results being generated

**Completed:**

- Execution finished
- Results available
- Output stored on Arweave
- Resulting messages created

**Failed:**

- Execution encountered error
- State not modified (or rolled back)
- Error message in output
- No resulting messages

#### Advanced Concepts

**Message Chains:**

- Messages that spawn other messages
- Form trees of related messages
- Track multi-step operations
- Analyze complex workflows

**Reference Tags:**

- Link messages to previous messages
- Create logical relationships
- Enable reply patterns
- Track conversation threads

**Cron Messages:**

- Scheduled recurring messages
- Automated process actions
- Background tasks
- Time-based triggers

**Spawning:**

- Process creating new processes
- Parent-child relationships
- Process factories
- Dynamic architecture

#### Common Patterns

**Query Pattern:**

```
Action: Eval
Data: return State.Variable
```

**Transfer Pattern:**

```
Action: Transfer
Recipient: [address]
Quantity: [amount]
```

**Custom Action Pattern:**

```
Action: Custom-Action
Param1: value1
Param2: value2
```

**Reply Pattern:**

```
Action: Reply
Ref_: [original-message-id]
Data: response-content
```

Remember: Messages are how everything happens in the AO network. Mastering message structure, actions, filtering, and analysis is crucial for effectively using Lunar and building on AO.
